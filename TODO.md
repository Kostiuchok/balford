# TODO — міграція balford.com.ua на WordPress

## Контекст проєкту
Мігруємо зі статичного сайту (`legacy-static/`) на WordPress — заради зручності
нетехнічного редактора контенту. Стейджинг: **https://balford.com.ua/wp-new/**
Мультимовність UA/EN — через **Polylang**, SEO — через **Yoast SEO**.
Планована версія PHP — **8.5**.

Хостинг: Mirohost, FTP-хост `wvh1.mirohost.net` (раніше фігурував як
`fvh16.mirohost.net` — актуальний саме `wvh1`), FTP root вже chroot-нутий
у домашню директорію (`/`, а не `/var/www/balford`), усередині — папка
`balford.com.ua/`, у ній `wp-new/`. Логін `balford`. FTP без TLS (сервер не
підтримує AUTH SSL/TLS — пароль передається у відкритому вигляді). Пароль
користувач тримає в себе, в цьому файлі не зберігається.

## ✅ Вирішено: версія PHP
PHP на сервері оновлено до **8.5.8** (підтверджено діагностикою 22.07.2026) —
ядро WordPress тепер вантажиться без fatal-помилок несумісності.

## ✅ Вирішено: доступ до бази даних

**Було:** підключення до MySQL падало з `Access denied for user
'u_blfrda'@'localhost' to database 'u_blfrda' (errno 1044)` — стара пара
юзер/база з `wp-config.php` була нечинною (грант відсутній / база з такою
назвою не існує).

**Виявлено (22.07.2026):** реальні робочі креденшели — юзер `u_balford` /
пароль `arkvgv6L` (наданий користувачем), прив'язаний до бази з іменем
**`balford`** (не `u_blfrda`). Ця база виявилась старою базою даних
**Joomla** (87 таблиць з префіксом `blf_*` — контент, jcomments, rsform
тощо), з тих часів, коли на цьому хостинг-акаунті стояв Joomla-сайт.

**Дія (за згодою користувача, без бекапу):** усі 87 `blf_*`-таблиць
видалено (`DROP TABLE`) через тимчасовий FTP-діагностичний скрипт —
база `balford` тепер порожня. `wp-config.php` на сервері оновлено новими
`DB_NAME=balford`, `DB_USER=u_balford`, `DB_PASSWORD=arkvgv6L`.

**Результат:** `https://balford.com.ua/wp-new/` тепер віддає **HTTP 302**
→ редірект на `wp-admin/install.php` (WP core повністю робочий, чекає на
майстер встановлення, бо `wp_*`-таблиць ще нема).

Користувач пройшов майстер встановлення сам (адмін-логін `admin`, пароль
відомий користувачеві, в пам'яті/репо не зберігається). Далі:
