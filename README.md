# Balford — balford.com.ua

Сайт компанії «Балфорд Україна» (альтернативна гідроенергетика, будівництво
малої ГЕС на річці Стрий, с. Довге-Гірське).

- **Продакшн-домен:** https://balford.com.ua/
- **Стейджинг:** https://balford.com.ua/wp-new/ (тимчасовий підкаталог,
  поки міграція не завершена й `/wp-new/` не стане кореневим сайтом)
- **Стек:** WordPress (кастомна тема `balford`) + **Polylang** (UA/EN) +
  **Yoast SEO**, shared-хостинг Mirohost (PHP 8.5, MySQL/MariaDB)

## Що це за проєкт

Міграція зі старого статичного HTML-сайту (лежить як референс у
`legacy-static/`, раніше на цьому ж хостинг-акаунті працював Joomla) на
WordPress — щоб нетехнічний редактор контенту міг сам оновлювати новини й
тексти сторінок через wp-admin, без втручання розробника.

Повна історія міграції, усі знайдені технічні проблеми та їх рішення,
поточний прогрес і план подальших робіт — у **`TODO.md`** в корені репо.
Цей файл (`README.md`) — довідник "як усе влаштовано і як розгорнути",
а не журнал прогресу; звіряйтеся з `TODO.md`, якщо потрібен актуальний
статус конкретно поточної сесії роботи.

## Структура репозиторію

```
legacy-static/          Стара статична версія сайту (HTML/CSS/JS) —
                         залишена як референс дизайну й оригінального
                         контенту (тексти новин, About/Ecology/Contacts
                         обома мовами). У продакшн не деплоїться.
wp-content/
  themes/
    balford/             Кастомна WordPress-тема — єдине, що версіюється
                          з боку WP. Усе інше (ядро WP, плагіни, uploads)
                          навмисно поза git — див. .gitignore нижче.
TODO.md                  Журнал міграції: що зроблено, що залишилось,
                          усі знайдені підводні камені.
README.md                Цей файл.
```

### Що НЕ в git і чому (`.gitignore`)

Версіюється лише код теми (`wp-content/themes/balford/**`). Поза git:
`wp-admin/`, `wp-includes/`, `wp-*.php`, `wp-config.php`, усі плагіни
(`wp-content/plugins/*`), `wp-content/uploads/`. Причина: сайт
розгортається поверх вже встановленого WordPress на хостингу — ядро й
плагіни ставляться напряму на сервері (з wordpress.org або через
wp-admin), а не через цей репозиторій. `wp-config.php` містить креденшели
БД і теж навмисно не версіюється.

## Хостинг і доступи

**Хостинг:** Mirohost (shared).
**FTP-хост:** `wvh1.mirohost.net` (без TLS — AUTH SSL/TLS сервер не
підтримує, пароль передається у відкритому вигляді; обмежте використання
довіреними мережами).
**FTP-логін:** `balford`.
**Структура на сервері:** FTP root вже chroot-нутий у домашню директорію
акаунта (тобто підключившись, ви одразу в `/`, а не десь у
`/var/www/...`). Усередині:
```
/                          ← FTP root (= chroot)
├── balford.com.ua/        ← публічний document root
│   ├── (файли старого legacy-static сайту, поки не замінені)
│   └── wp-new/             ← стейджинг WordPress (поки тут, не в корені)
│       ├── wp-config.php
│       ├── wp-admin/, wp-includes/  ← ядро WP
│       └── wp-content/
│           ├── themes/balford/      ← ця тема
│           └── plugins/             ← Polylang, Yoast SEO, Site Kit
├── cgi-bin/
└── tmp/
```

**PHP:** 8.5 (оновлено з застарілої 5.4, яка стояла на акаунті до старту
міграції — стежте, щоб хостинг-панель не відкотила версію).

**База даних:** MySQL/MariaDB (11.8.8-MariaDB на момент міграції), назва
бази й окремий БД-юзер — **не `wp_...` за замовчуванням**, актуальні
значення дивіться в `wp-config.php` на сервері (не в репо). БД раніше
використовувалась старим Joomla-сайтом на цьому ж акаунті — весь
Joomla-контент (87 таблиць `blf_*`) видалено під час міграції, база
перевикористана під WordPress з нуля.

### Де реально лежать паролі

**Жодного пароля/секрету немає в цьому репозиторії й не повинно бути.**
FTP-пароль, БД-пароль, пароль адміністратора wp-admin — тримає власник
проєкту окремо (менеджер паролів чи інший приватний канал). Якщо
працюєте з цим проєктом (людина чи асистент) — попросіть креденшели
безпосередньо у власника на початку сесії; не зберігайте їх у файлах
репозиторію, конфігах, коментарях коду чи будь-де, звідки вони можуть
потрапити в git.

## Як розгорнути WordPress з нуля (якщо стейджинг колись пересоздається)

1. Завантажити WordPress core на сервер (FTP або хостинг-панель), в
   потрібну директорію (`wp-new/` на стейджингу, або корінь
   `balford.com.ua/` для продакшена).
2. Створити `wp-config.php` з даними підключення до БД (сама база й юзер
   створюються через панель керування хостингом — розділ MySQL databases;
   переконайтесь, що юзера явно **прив'язано (grant)** до бази, це
   окремий крок від просто створення юзера, легко забути).
3. Пройти майстер встановлення WP (`/wp-admin/install.php`) — назва
   сайту, адмін-логін/пароль, email.
4. Скопіювати вміст `wp-content/themes/balford/` із цього репо на сервер
   у `wp-content/themes/balford/`, активувати тему (Зовнішній вигляд →
   Теми).
5. Встановити й активувати плагіни: **Polylang** (безкоштовна версія з
   wordpress.org достатньо), **Yoast SEO**.
6. Налаштувати мови в Polylang: **uk** — за замовчуванням, **en** —
   друга. **Важливо:** локаль української в довіднику Polylang — просто
   `uk`, НЕ `uk_UA` (з `uk_UA` мова не додається, "invalid characters").
7. Увімкнути CPT `news` для перекладу: за замовчуванням кастомні типи
   постів НЕ перекладаються Polylang, поки явно не додати в
   Мови → Налаштування → Спеціальні типи записів і таксономії →
   позначити `news`. Без цього кроку архів `/news/` показує впереміш
   обидві мови, а `/en/news/` віддає 404.
8. Після зміни налаштувань перекладу типів записів — зайти в
   Налаштування → Постійні посилання і натиснути "Зберегти" (це форсує
   `flush_rewrite_rules()`; без цього нові мовні URL для `news` не
   запрацюють, навіть якщо крок 7 виконано).
9. Перенести контент: сторінки About/Ecology/Contacts (обома мовами,
   тексти є в `legacy-static/index_ua.html` і `index_en.html`) і новини
   (з окремих `legacy-static/news-*.html` файлів, зображення — з
   `legacy-static/img/news/`, теж є копії в `wp-content/themes/balford/assets/img/news/`).

## Особливості теми `balford` (для розробника)

- **Custom Post Type `news`** реєструється в `functions.php`
  (`balford_register_news_cpt()`), архів за адресою `/news/`.
- **Сторінки з власним шаблоном за slug:** `page-about.php`,
  `page-ecology.php`, `page-contacts.php` підвантажуються автоматично
  для сторінок зі slug `about`/`ecology`/`contacts` (стандартна
  WP template hierarchy). **Для мовних перекладів slug інший**
  (`about-company`, `ecology-impact`, `contact-us` для EN — Polylang не
  дозволяє однаковий slug для двох мов "з коробки"), тому для EN-сторінок
  шаблон прописаний явно через постмету `_wp_page_template`, а не
  покладається на збіг slug.
- **Локалізація UI-рядків**, що не прив'язані до конкретного посту/сторінки
  (hero-текст, підписи кнопок, запасне меню, футер) — через обгортку
  `balford__()` над `pll__()` Polylang (`functions.php`). Рядки
  реєструються `pll_register_string()`, видно в
  Мови → Переклад рядків для ручного редагування за потреби.
- **`balford_translated_page( $uk_slug )`** (`functions.php`) — хелпер
  для отримання правильної мовної версії сторінки за її uk-slug (бо
  `get_page_by_path()` сам по собі мову не враховує). Використовується і
  в запасному меню, і в тизерах на головній — без нього посилання й
  контент на EN-сторінках показували б українську версію.
- **Хлібні крихти** — через `yoast_breadcrumb()`, обгорнуто в
  `balford_breadcrumbs()`. Потребує увімкненої опції
  "Breadcrumbs" в Yoast SEO (SEO → Search Appearance → Breadcrumbs).
- Верстка — Bootstrap 3 + власний `pheromone.css`/`pheromone.js`
  (спадщина від оригінального шаблону, з якого зроблено legacy-static
  сайт). jQuery, не React/Vue — це класична PHP-тема без збірки/бандлера.

## Локальна розробка

Проєкт не має локального dev-сервера/докера в репозиторії — редагування
файлів теми відбувається прямо в цій папці, а деплой на сервер зараз
робиться вручну через FTP (по одному файлу чи через FTP-клієнт). Якщо
потрібне автоматизоване локальне середовище (Local by Flywheel, Docker,
`wp-env` тощо) — не налаштовано, можна додати за потреби.

Щоб побачити зміни теми наживо без повного локального WP:
1. Відредагувати потрібний файл у `wp-content/themes/balford/`.
2. Залити той самий файл на сервер у
   `wp-new/wp-content/themes/balford/` через FTP.
3. Перевірити на стейджингу (`https://balford.com.ua/wp-new/`).

## Мультимовність (Polylang) — коротко

- UA — мова за замовчуванням (URL без префіксу: `/about/`, `/news/...`).
- EN — з префіксом `/en/` (`/en/about-company/`, `/en/news/...-en`).
- Перемикач мов у навбарі — dropdown "Укр"/"Eng" (виводиться в
  `balford_default_menu()` через `pll_the_languages()`).
- Переклади сторінок/новин зв'язуються через Polylang (кожна мовна версія
  — окремий WP-пост/сторінка, зв'язані як "переклади один одного").
